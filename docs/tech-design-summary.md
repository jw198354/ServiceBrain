# 技术方案设计摘要

## 核心设计原则

1. **轻量单体优先** - 避免过早引入重型微服务和 MQ
2. **核心链路优先** - 优先打通主链路
3. **受控式 AI** - 模型负责理解与生成，规则层控制高风险决策
4. **状态可见** - 前端页面状态、会话状态、连接状态都要可追踪
5. **记忆外部化** - Prompt 只装载必要上下文，历史以摘要和检索方式使用
6. **Demo 可交付 + 产品化可扩展** - 接口协议、状态模型、模块分层不做死

## 关键技术决策

| 问题 | 方案 |
|------|------|
| 匿名接入 | 前端本地 token + 后端匿名用户模型 |
| 实时通信 | WebSocket (非轮询) |
| 意图识别 | 一次模型调用输出结构化结果 |
| 补槽追问 | 会话状态维护待补字段 |
| 咨询 vs 执行 | 编排层判定，非关键词路由 |
| 知识 vs 解释 | RAG Answer Chain vs RAG+Context Explain Chain |
| Tool 调用 | 编排层校验，非模型自由调用 |
| 记忆管理 | 工作记忆 + 摘要检索，非全量历史 |

## 核心业务链路

```
用户进入 → 匿名身份检查 → 用户名采集 → 会话初始化 → WebSocket 建连
                                      ↓
                              机器人首问 + 快捷问题
                                      ↓
    ┌─────────────────────────────────┼─────────────────────────────────┐
    ↓                                 ↓                                 ↓
【退款咨询】                    【退款执行】                      【知识答疑】
用户提问 → 判断是否需要          用户请求 → 补参数 →              纯规则问题
订单号 → 规则解释               规则校验 → Tool 调用 →            → RAG 检索
                               结果卡片                        → 结构化文本
```
